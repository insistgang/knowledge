## 一、计算机视觉基础方法

### **1. 图像增强 (Image Enhancement)** - CLAHE算法

```python
def enhance_dark_regions(image):
    """增强暗部区域"""
    # 1. 转换到LAB色彩空间
    lab = cv2.cvtColor(image, cv2.COLOR_BGR2LAB)
    l, a, b = cv2.split(lab)
    
    # 2. 对亮度通道应用CLAHE
    clahe = cv2.createCLAHE(clipLimit=3.0, tileGridSize=(8,8))
    l_enhanced = clahe.apply(l)
    
    # 3. 转回BGR
    enhanced_lab = cv2.merge([l_enhanced, a, b])
    enhanced_bgr = cv2.cvtColor(enhanced_lab, cv2.COLOR_LAB2BGR)
```

**技术名称**：**CLAHE (Contrast Limited Adaptive Histogram Equalization)**
- **论文出处**：Pizer et al. "Adaptive Histogram Equalization and Its Variations" (1987)
- **核心思想**：局部自适应直方图均衡化，增强暗部细节
- **参数说明**：
  - `clipLimit=3.0`：限制对比度增强幅度，防止噪声放大
  - `tileGridSize=(8,8)`：将图像分成8×8的小块，分别增强

**为什么在LAB空间**：
- L通道：亮度（Lightness）
- A通道：红绿色彩（Green-Red）
- B通道：黄蓝色彩（Blue-Yellow）
- **优势**：只调整亮度，不影响颜色，更符合人眼视觉

**效果**：
```
暗光场景: [亮度分布 20-60] → CLAHE → [亮度分布 40-180]
         看不清路灯          →        路灯清晰可见
```

### **2. 颜色直方图特征 (Color Histogram)** - ReID基础

```python
def extract_feature(self, image, bbox):
    # 提取ROI
    roi = image[y1:y2, x1:x2]
    
    # 计算3通道颜色直方图
    hist_features = []
    for i in range(3):  # BGR三通道
        hist = cv2.calcHist([roi], [i], None, [32], [0, 256])
        hist = cv2.normalize(hist, hist).flatten()
        hist_features.extend(hist)
    
    # L2归一化
    feature = np.array(hist_features, dtype=np.float32)
    feature = feature / (np.linalg.norm(feature) + 1e-6)
```

**技术名称**：**Color Histogram (颜色直方图特征)**
- **经典方法**：Swain & Ballard "Color Indexing" (1991)
- **特征维度**：3通道 × 32bins = 96维
- **归一化**：L2 norm使特征模长为1，便于余弦相似度计算

**为什么用32个bin**：
- 太少（如8）：特征太粗糙，区分能力弱
- 太多（如128）：对光照变化敏感，过拟合
- **32是经验最优值**

**相似度计算**：
```python
similarity = np.dot(feat1, feat2)  # 余弦相似度
# 因为已经L2归一化，点积 = cosθ ∈ [-1, 1]
```

## 二、目标跟踪核心算法

### **3. IoU (Intersection over Union)** - 位置匹配

```python
def calculate_iou(self, box1, box2):
    # 计算交集区域
    inter_x_min = max(x1_min, x2_min)
    inter_y_min = max(y1_min, y2_min)
    inter_x_max = min(x1_max, x2_max)
    inter_y_max = min(y1_max, y2_max)
    
    inter_area = (inter_x_max - inter_x_min) * (inter_y_max - inter_y_min)
    
    # 计算并集
    union_area = box1_area + box2_area - inter_area
    
    # IoU = 交集/并集
    return inter_area / union_area
```

**技术名称**：**IoU (Intersection over Union)**
- **论文出处**：Everingham et al. "The PASCAL Visual Object Classes Challenge" (2010)
- **核心公式**：
  $$IoU = \frac{Area(box_1 \cap box_2)}{Area(box_1 \cup box_2)}$$
- **取值范围**：[0, 1]，1表示完全重叠

### **4. 中心距离 (Center Distance)** - 空间约束

```python
def calculate_center_distance(self, box1, box2):
    # 计算框中心点
    cx1 = (x1_min + x1_max) / 2
    cy1 = (y1_min + y1_max) / 2
    cx2 = (x2_min + x2_max) / 2
    cy2 = (y2_min + y2_max) / 2
    
    # 欧氏距离
    return np.sqrt((cx1 - cx2)**2 + (cy1 - cy2)**2)
```

**技术名称**：**Euclidean Distance (欧氏距离)**
- **数学基础**：欧几里得几何
- **核心公式**：
  $$d = \sqrt{(x_1-x_2)^2 + (y_1-y_2)^2}$$
- **物理意义**：两点间直线距离

**为什么同时用IoU和距离**：
- **IoU**：擅长判断重叠（近距离）
- **距离**：擅长判断远距离物体（IoU=0时仍可用）
- **互补性**：一个失效时另一个仍有效

### **5. 加权相似度融合 (Weighted Similarity)** - 多准则决策

```python
def calculate_similarity(self, box1, box2):
    distance = self.calculate_center_distance(box1, box2)
    iou = self.calculate_iou(box1, box2)
    
    # 只有满足阈值才计算
    if distance < self.distance_threshold or iou > self.iou_threshold:
        distance_score = max(0, 1 - distance / 200)
        # 加权融合：60% IoU + 40% 距离
        return 0.6 * iou + 0.4 * distance_score
    return 0.0
```

**技术名称**：**Multi-Criteria Decision Making (多准则决策融合)**
- **理论基础**：加权平均法 (Weighted Average Method)
- **权重设计**：
  - **IoU权重0.6**：位置重叠更重要（近距离精确匹配）
  - **距离权重0.4**：补充远距离判断
- **归一化**：`distance_score = 1 - distance/200` 将距离映射到[0,1]

**为什么200像素**：
- 典型1920×1080视频中，路灯间距约50-150像素
- 200像素是安全裕度，超过则认为不是同一路灯

### **6. 匈牙利算法 (Hungarian Algorithm)** - 最优匹配

```python
try:
    from scipy.optimize import linear_sum_assignment
    
    # 构造代价矩阵（相似度→代价）
    cost_matrix = 1 - similarity_matrix
    cost_matrix[similarity_matrix < 0.1] = 1e6  # 低相似度设为无穷大
    
    # 求解最优分配
    row_indices, col_indices = linear_sum_assignment(cost_matrix)
```

**技术名称**：**Hungarian Algorithm / Kuhn-Munkres Algorithm (匈牙利算法)**
- **论文出处**：Kuhn "The Hungarian Method for the assignment problem" (1955)
- **问题类型**：二分图最优匹配（Assignment Problem）
- **时间复杂度**：O(n³)，scipy实现优化到O(n²·log(n))

**核心思想**：
```
旧对象:  A    B    C       新检测:  1    2    3

相似度矩阵:
      1    2    3
  A  0.9  0.2  0.1
  B  0.3  0.8  0.2
  C  0.1  0.3  0.9

代价矩阵 (1 - 相似度):
      1    2    3
  A  0.1  0.8  0.9   →  最优匹配: A-1, B-2, C-3
  B  0.7  0.2  0.8       总代价最小 = 0.1+0.2+0.1 = 0.4
  C  0.9  0.7  0.1
```

**为什么比贪心好**：
- **贪心算法**：每次选最大相似度，可能陷入局部最优
- **匈牙利算法**：全局最优解，避免ID混乱

### **7. ReID重链接 (Re-Identification)** - 特征匹配

```python
def find_relink_candidate(self, new_feature, new_class, excluded_ids=None):
    best_match_id = None
    best_similarity = 0
    
    # 遍历消失的对象
    for obj_id in list(self.disappeared.keys()):
        if obj_id in excluded_ids:  # 避免ID重复
            continue
        
        # 只在同类别中查找
        if self.id_class_history.get(obj_id) != new_class:
            continue
        
        # 计算特征相似度
        similarity = self.calculate_feature_similarity(
            new_feature, 
            self.object_features[obj_id]
        )
        
        # 超过阈值则认为是同一对象
        if similarity > best_similarity and similarity > self.feature_threshold:
            best_similarity = similarity
            best_match_id = obj_id
    
    return best_match_id, best_similarity
```

**技术名称**：**Re-Identification (重识别/重链接)**
- **论文出处**：DeepSORT "Simple Online and Realtime Tracking with a Deep Association Metric" (2017)
- **核心思想**：通过外观特征恢复被遮挡后重新出现的对象
- **关键约束**：
  1. **类别一致性**：只在同类别中查找（OFF不会匹配ON）
  2. **特征相似度阈值**：默认0.7，高阈值避免误匹配
  3. **ID唯一性**：`excluded_ids` 防止重复使用

**应用场景**：
```
帧100: ID=5 路灯被车遮挡 → disappeared[5] = 1
帧101: 无检测              → disappeared[5] = 2
帧102: 车离开，检测到路灯  → ReID匹配到ID=5，恢复跟踪
```

## 三、统计学与投票方法

### **8. 多数投票 (Majority Voting)** - 类别稳定性

```python
def get_most_common_class(self, obj_id):
    """获取出现最多的类别（多数投票）"""
    if obj_id in self.class_history and len(self.class_history[obj_id]) > 0:
        # 统计频次，返回最多的类别
        most_common = max(set(self.class_history[obj_id]), 
                        key=self.class_history[obj_id].count)
        return most_common
```

**技术名称**：**Majority Voting (多数投票/投票法)**
- **理论基础**：集成学习 (Ensemble Learning)
- **论文出处**：Dietterich "Ensemble Methods in Machine Learning" (2000)
- **核心公式**：
  $$class_{final} = \arg\max_{c} \sum_{i=1}^{n} \mathbb{1}(class_i = c)$$

**实际案例**：
```python
class_history[ID=10] = [0, 0, 1, 0, 1, 0, 0, 0]
#                       ↑  ↑  ↑  ↑  ↑  ↑  ↑  ↑
#                      帧1 帧2 帧3 帧4 帧5 帧6 帧7 帧8

# 统计：0出现6次，1出现2次
# 投票结果：final_class = 0 (OFF)
```

**为什么有效**：
- **抗短时误检**：车灯短暂干扰（帧3、帧5）不影响最终结果
- **提升鲁棒性**：单次错误不会影响整体判断
- **平滑时间序列**：消除噪声波动

### **9. 四维稳定性过滤 (Multi-Criteria Filtering)** - 质量保证

```python
def is_stable(self, obj_id):
    # 维度1: 检测次数
    if hits < self.min_hits:
        return False, "检测次数不足"
    
    # 维度2: 存活时间
    lifespan = last_frame - first_frame + 1
    if lifespan < self.min_frames:
        return False, "存活时间不足"
    
    # 维度3: 平均置信度
    avg_conf = np.mean(self.confidence_history[obj_id])
    if avg_conf < self.min_avg_confidence:
        return False, "置信度过低"
    
    # 维度4: 位置合理性
    avg_y = np.mean([y for y1, y2 in position_history])
    if avg_y > self.max_y_position:
        return False, "位置异常"
    
    return True, "稳定"
```

**技术名称**：**Multi-Dimensional Filtering (多维度过滤)**
- **理论基础**：AND逻辑组合（所有条件同时满足）
- **四个维度**：

| 维度 | 技术方法 | 论文依据 | 物理意义 |
|------|---------|---------|---------|
| 检测次数 | Hit Count Filter | SORT (2016) | 稳定性：多次检测=真实目标 |
| 存活时间 | Lifespan Filter | 经验阈值 | 持久性：长时存在=非噪声 |
| 平均置信度 | Confidence Filter | Statistical Thresholding | 质量：高置信度=可靠检测 |
| 位置合理性 | Spatial Constraint | Domain Knowledge | 先验知识：路灯在高处 |

**逻辑表达式**：
$$Stable = H \land L \land C \land P$$
其中：
- $H$: hits ≥ min_hits
- $L$: lifespan ≥ min_frames  
- $C$: avg_conf ≥ min_avg_confidence
- $P$: avg_y ≤ max_y_position

**效果对比**：
```
无过滤:   100个检测 → 包含误检、噪声
单维过滤:  80个检测 → 仍有部分误检
四维过滤:  52个检测 → 高质量、稳定检测 ✓
```

## 四、数据结构与算法设计

### **10. 状态机管理 (State Machine)** - 生命周期跟踪

```python
# 状态转换
初始化 → 检测到 → 匹配成功 → 更新状态 → 稳定/不稳定
                        ↓
                    匹配失败 → disappeared++ → 超时删除
```

**核心数据结构**：
```python
self.objects = {}           # 当前活跃对象
self.disappeared = {}       # 消失计数器
self.stable_ids = set()     # 稳定对象集合
self.active_ids_in_frame = set()  # 当前帧活跃ID（防重复）
```

**状态转换表**：
| 当前状态 | 事件 | 下一状态 | 操作 |
|---------|------|---------|------|
| 新检测 | - | 创建 | `next_id++`, 初始化历史 |
| 跟踪中 | 匹配成功 | 跟踪中 | 更新位置、特征 |
| 跟踪中 | 匹配失败 | 消失 | `disappeared++` |
| 消失 | ReID成功 | 跟踪中 | 恢复跟踪 |
| 消失 | 超时 | 删除 | 评估稳定性，删除ID |
