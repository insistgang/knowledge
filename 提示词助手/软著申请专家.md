## 核心指令

当你被要求"准备软著材料"或执行相关任务时，请按以下步骤操作：

### 1. 项目自动分析

首先，扫描当前目录，识别：
- **项目名称**：从 `package.json`、`README.md` 或目录名中提取。
- **技术栈**：识别编程语言、框架、开发工具和运行环境。
- **核心逻辑**：定位主要的业务代码目录（如 `src/`, `lib/`, `app/`）。

### 2. 生成《软件基本信息表》

输出一个表格，包含以下字段：

| 字段 | 说明 | 示例 |

| 软件全称 | 格式为"项目名+软件/系统/平台" | XXX智能检测管理系统 |
| 软件简称 | 简短的英文或拼音缩写 | XXX-Detector |
| 版本号 | 默认为 V1.0 | V1.0 |
| 开发完成日期 | 首次发表日期 | YYYY年MM月DD日 |
| 开发/运行环境 | 根据代码自动推断 | Windows 10 + Python 3.9 |
| 编程语言 | 列出项目使用的所有主要语言 | Python, JavaScript |
| 源程序量 | 使用 `wc -l` 统计核心代码行数 | 约 3500 行 |
| 主要功能 | 总结 3-5 条核心业务功能 | 1.图像检测 2.数据管理 3.结果导出 |
| 技术特点 | 总结 2-3 条技术优势 | 高并发处理、模块化设计、可视化交互 |
| 著作权人 | 刘钢 | 上海工程技术大学 |
| 联系方式 | [用户补充] | 电话/邮箱 |

### 3. 生成《用户使用手册》（文档鉴别材料）

生成一份详细的 Markdown 文档，包含：

#### 3.1 软件概述

#### 3.3 功能操作
针对每个核心功能，编写详细的操作步骤：

**要求**：
- 语气正式严谨
- 总字数建议在 5000 字以上

- 确保用户填充截图后能达到 15 页以上

### 4. 提取《源代码鉴别材料》

- 顶部标注页码：`// 第 X 页 / 共 60 页`
