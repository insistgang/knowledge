## 环境配置

### 多平台支持
- Windows 本机
- WSL (Windows Subsystem for Linux)
- Ubuntu 虚拟机

### 配置要点
- 配置文件位置: `~/.claude/` 或项目 `.claude/`
- 关键文件: `CLAUDE.md` (项目上下文), `settings.json`, `.env`

### API 配置
支持多种 AI 供应商切换：

| 供应商 | 用途 | 备注 |
|--------|------|------|
| Kimi | 日常主力 | 上下文能力强 |
| GLM | 后端任务 | 智谱清言 |
| AnyRouter | Claude代理 | 需梯子 |
| HaoZCloud | AI Gate | 国内直连 |
| OpenRouter | 346+模型 | 需梯子 |
| GPTsAPI | Azure | 稳定 |
| xiavier | Claude4.5全系列 | 最新模型 |

```bash

ls -la | tree

# 查找文件
find . -name "*.md"

# 批量重命名
```

## 问题解决

### GitHub 推送失败
- 问题: 代理导致 SSH 连接不稳定
- 解决: 从 SSH 切换到 HTTPS，跳跃代理

### 图片无法打开
- 问题: Clawdbot 修复后出现路径问题
- 解决: 检查文件权限，使用绝对路径

### IP 混乱 (WSL)
- 问题: WSL 网络 IP 配置冲突
- 解决: 重置网络配置，重启 WSL

## 最佳实践

### 文件命名
- 统一使用 `YYYY-MM-DD` 格式
- 避免特殊字符
- 英文命名为主，中文注释为辅

### 备份策略
- GitHub 远程备份
- 定期 commit
- 重要文件多重备份

### 安全注意
- API Key 使用环境变量
- `.env` 文件不提交到 Git
- 敏感信息单独管理
