# 详细审稿意见

## 论文信息

- **标题**: VSSU3D: 3D Visual State Space U-Net for Denoising on Hyperspectral Image
- **领域**: 高光谱图像去噪、深度学习

------

## 总体评价

本文提出将Mamba模型的状态空间思想与3D卷积、U-Net架构相结合，用于高光谱图像去噪。研究方向有一定价值，但论文在**创新性论证、技术细节完整性、实验严谨性**等方面存在较多问题，需要进行较大幅度的修改。

**推荐意见：大修 (Major Revision)**

**评分（满分5分）：**

- 创新性: 3.0/5
- 技术质量: 2.5/5
- 实验完整性: 2.5/5
- 写作质量: 3.0/5
- 总体: 2.75/5

## 第一部分：创新性与贡献度分析

### 1.1 创新点界定模糊

**问题1：与Vision Mamba的本质区别不清晰**

论文在3.2.4节声称与现有Mamba模型有"fundamental differences"，但论证不够充分：

- "Hybrid 3D-Sequence Modeling"：将光谱维度作为序列轴并非全新思路，VMamba、Vim等工作已有类似探索
- "Dynamic State Space Parameters"：这正是原始Mamba的selective mechanism，作者需要说明具体做了哪些针对HSI的改进
- 缺少与ViM (Vision Mamba)、VMamba、LocalMamba等最新视觉Mamba变体的详细对比

**建议**：

- 增加表格明确对比本文VSS与至少3种现有视觉Mamba变体的技术差异
- 提供理论或实验证据说明这些差异对HSI去噪的具体增益

**问题2：SSCA模块创新性有限**

SSCA本质上是标准的SE-Net通道注意力机制的3D版本，主要改动是：

- 将2D卷积替换为3D卷积
- 简化了部分结构

这种改进属于**增量式创新**，建议作者：

- 与SE3D、CBAM3D等进行对比实验
- 说明"简化"具体简化了哪些部分，参数量/计算量减少多少

### 1.2 贡献陈述过度

论文声称的三点贡献：

| 声称的贡献                                               | 实际情况                                    | 建议                                      |
| -------------------------------------------------------- | ------------------------------------------- | ----------------------------------------- |
| "superior denoising performance compared to SOTA"        | 仅在2/3数据集上最优，CAVE上明显落后         | 措辞应更准确，如"competitive performance" |

## 第二部分：技术内容深度分析

### 2.1 理论基础问题

**问题3：状态空间模型扩展的数学严谨性不足**

公式(2)的状态空间方程： $$h_t = A_t h_{t-1} + B_t x_t$$ $$y_t = C_t h_t + D_t x_t$$

**存在的问题**：

- $h_t$的维度是多少？状态空间的大小如何确定？
- 离散化过程未说明（Mamba使用ZOH离散化）
- $A_t$如何保证稳定性？是否继承了HiPPO矩阵的结构？

公式(3)中： $$A_t = \exp(\Delta_t \cdot A_{log})$$

- $\Delta_t$是如何计算的？原始Mamba中$\Delta$是通过softplus激活得到的，本文是否相同？
- $A_{log}$的初始化细节？

**建议**：补充完整的数学推导，特别是：

- 状态空间维度的选择依据
- 离散化方法
- 参数初始化策略

**问题4：计算复杂度分析缺失**

论文多次提及"significantly reduce computational complexity"、"linear computational complexity"，但：

- 未提供理论复杂度分析（如O(n)具体是相对于什么的n？）
- 未与Transformer的O(n²)进行定量对比
- Table I中本文GFLOPs (35.30) 反而高于多数对比方法

**建议**：增加专门的复杂度分析小节，包括：

- 理论时间复杂度
- 理论空间复杂度
- 不同输入尺寸下的实际运行时间曲线

### 2.2 网络架构细节问题

图1中标注了"Bidirectional"和"Alternating Directional"，但正文中完全没有解释：

## 第三部分：实验设计与分析

### 3.1 实验设置问题

**问题9：噪声模型过于简化**

论文仅使用加性高斯白噪声(AWGN)，但真实HSI噪声特性包括：

- 信号相关噪声(signal-dependent noise)
- 条纹噪声(stripe noise)
- 脉冲噪声(impulse noise)
- 混合噪声

作者自己在第13页承认CAVE数据集的条纹噪声导致性能下降，这说明**模型的泛化能力存在问题**。

**建议**：

- 增加混合噪声实验
- 增加真实噪声数据集测试（如EO-1 Hyperion）
- 或在limitations中明确说明适用范围

**问题10：噪声水平(σ)未报告**

这是**严重缺陷**。不同噪声强度下的性能是评估去噪算法的基本要求：

- σ = 10, 30, 50, 70等不同级别的PSNR/SSIM应分别报告
- 应包含盲去噪(blind denoising)的实验

**问题11：数据集划分问题**

| 数据集       | 划分方式          | 问题                           |
| ------------ | ----------------- | ------------------------------ |
| ICVL         | 20/5 (train/test) | 无验证集，如何early stopping？ |
| HOUSTON 2018 | 70/15/15随机划分  | 是否多次随机划分取平均？       |
| CAVE         | 70/15/15随机划分  | 同上                           |

随机划分可能导致结果variance较大，建议：

- 报告多次实验的均值±标准差
- 或使用固定的公开划分

**问题12：Patch-based评估的局限性**

- 训练和测试都使用64×64 patches
- 未报告全图推理的性能
- Patch边界的处理方式未说明（overlap? blending?）

### 3.2 对比方法问题

**问题13：对比方法时效性不足**

| 方法     | 年份 | 类型            |
| -------- | ---- | --------------- |
| DnCNN    | 2017 | CNN             |

| QRNN3D   | 2020 | RNN             |

**缺少的重要对比方法**：

- 通用图像去噪的最新方法在HSI上的表现（如Restormer, NAFNet等）

**问题14：对比不公平的可能性**

- 是否所有方法都使用相同的训练数据和增强策略？
- 是否都训练到收敛？
- 对比方法的超参数是否经过调优？

- 在什么场景/噪声类型下模型表现最差？

## 给作者的建设性建议

1. **聚焦核心贡献**：建议将重点放在"如何将SSM有效适配到3D HSI数据"这一问题上，而非宣称全面超越SOTA
2. **完善实验**：目前实验最大的问题是噪声模型单一和缺少实现细节，这直接影响论文的实用价值和可复现性
3. **诚实对待不足**：在CAVE上的表现说明模型对非高斯噪声的泛化能力有限，建议在正文中增加limitations小节讨论
4. **增强理论深度**：当前论文偏工程实现，缺少对"为什么这样设计有效"的深入分析，建议增加理论动机和分析
5. **提升可视化**：当前图表质量一般，好的可视化能大幅提升论文的说服力

## 最终评价

本文选题有一定价值，将新兴的State Space Model引入HSI去噪是有意义的探索。但目前版本存在较多问题：技术细节不完整影响可复现性，实验设计不够严谨影响结论可信度，与最新相关工作的对比不足影响贡献度评估。建议作者进行大幅修改后重新投稿。
