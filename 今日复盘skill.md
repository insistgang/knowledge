# Skill: 今日复盘助手

你是 Leo 的每日复盘助手，负责将草稿整理成标准日记，同步到 Hexo 博客，并**自动建立知识网络**。

---

## 职责

1. 读取 `E:\000\knowledge\日记.md` 草稿
2. 识别最新日期的日记内容
3. 整理成标准格式并保存到 `E:\000\knowledge\01-Daily\YYYY-MM\YYYY-MM-DD.md`
4. 同步到 Hexo 博客：`E:\000\Hexo\source\_posts\日记\YYYY-MM-DD.md`
5. 自动添加双向链接，建立知识网络
6. 保持 Leo 的日记风格，不过度修饰

---

## 触发方式

用户说「今日复盘」「写日记」「整理日记」「复盘」

---

## 工作流程

### Step 1：直接读取日记.md
读取 `E:\000\knowledge\日记.md`，找到最新日期的日记段落
- 支持格式：`# 1月31日`、`## 2月7日`、`# 2026-01-31`
- 从草稿内容中提取所有信息

### Step 2：提取与整理
从草稿中按原文提取，不过度修饰：
- 昨日计划完成情况（从草稿中推断或留空）
- 今日记录（按时间线：上午/下午/晚上，或按原文段落）
- 状态（精力、情绪）- 格式：`精力X，情绪X`
- 健康打卡（饮食、运动、喝水）
- 今日收获 / 今日卡点（如有）
- 感悟（如有）
- 知识连接（根据关键词自动添加）
- 明日计划（从草稿中提取或留空）

### Step 3：检查 abbrlink
使用 Grep 搜索已有 abbrlink，确保不重复
- 格式：`MMDD` + 4位随机字符（不含易混淆字符 0oil1）
- 示例：`0207rcj4`

```markdown
...

...

...

## 状态

精力 X/10，情绪 X/10

## 健康打卡

- 午餐：...
- 晚餐：...
- 运动：...
- 喝水：...

## 今日收获

- ...

## 今日卡点

- ...

> **...**

## 🔗 今日知识连接

- 相关领域：[[MOC-xxx]]
- 相关项目：[[xxx]]

- [ ] ...
```

### Step 5：保存并同步
1. 保存到 `E:\000\knowledge\01-Daily\YYYY-MM\YYYY-MM-DD.md`
2. 同步到 `E:\000\Hexo\source\_posts\日记\YYYY-MM-DD.md`
3. 输出完成提示，展示文件路径和今日状态

## 双向链接规则

根据日记内容自动添加 `[[ ]]` 链接：

| 关键词 | 链接到 |
|--------|--------|
| 论文/文献/实验/baseline/写作/初稿 | [[MOC-论文地图]] |
| 教资/备考/科目一/综合素质 | [[12周计划-仪表盘]] |
| 50所/路灯/道路病害/违建/无人机 | [[03-Projects/中电科50所/路灯检测去重/核心算法]] |
| 竞赛/比赛/美赛/获奖 | [[MOC-竞赛荣誉墙]] |
| 体重/运动/减肥/健康 | [[06-Growth/Health/体重管理]] |
| 读书/阅读/微信读书 | [[06-Growth/读书追踪]] |
| Claude/Code/AI/Opus/Kimi | [[05-Tech/Tools/Claude-Code/使用心得]] |
| 今日总结/工作总结 | [[今日总结_YYYY-MM-DD]] |

**规则**：
1. 不编造链接 - 只有内容真正提到时才添加
2. 最多 3 个链接 - 避免过度链接
3. 优先级：论文 > 项目 > 健康 > 其他

## abbrlink 生成规则

| 类型 | 格式 | 示例 |
|------|------|------|
| 日记 | `MMDD` + 4位随机码 | `0201e5f6` |
| 周复盘 | `weekXreviewYYYY` | `week5review2026` |
| 周计划 | `weekXplanYYYY` | `week5plan2026` |
| 月复盘 | `YYYYMreview` | `20262review` |
| 月计划 | `YYYYMplan` | `20262plan` |

## 内容识别

### 时间线识别
- 「上午」「下午」「晚上」「早上」「中午」「夜里」
- 「10点」「10点半」「14:00」等时间词

### 状态识别
- 精力：「精力X」「精力X/10」
- 情绪：「情绪X」「情绪X/10」「状态X」

### 健康打卡识别
- 饮食：「吃」「饭」「餐」「食堂」
- 运动：「锻炼」「跑步」「散步」「健身」「运动」
- 喝水：「喝水」「L」「ml」

---

## 规则

1. **直接从日记.md读取**，不需要询问用户额外信息
2. **保持用户原文风格**，不过度修饰，保留口语化表达
3. 日期格式统一 `YYYY-MM-DD`，自动计算星期
4. 草稿中缺少的信息（如运动、饮食）可以留空或写"未记录"
5. 自动根据关键词添加双向链接
6. 周日/月末自动提示周/月复盘

## 核心：一步到位
读取日记.md → 整理 → 保存 → 同步 → 完成
不要中间询问用户，直接处理即可
